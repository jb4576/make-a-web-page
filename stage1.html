HTML Key Concepts

Lesson 1: Foundations of the Web and HTML

How the Web Works

The web is a collection of computers, servers and devices that communicate using a variety of standards. The most common standard for web pages is the the HTML (Hypertext Markup Language) standard with the latest common release being HTML5. These pages are commonly viewed on web browsers such as Internet Explorer, Firefox and Chrome. Most web pages are delivered over TCP port 80 when performing an HTTP request.

A good resource for the basics on HTML can be found at w3schools.com

Elements

HTML documents are made up of various HTML elements that have a start and an end tag with content in between. The elements can work without an end tag but that element property will apply to all content after the start tag. An end tag is recommended regardless if it is required.

A good reference for elements can be found at w3schools.com

Attributes

HTML attributes provide additional details in reference to HTML elements. These attributes can be an element value or provide additional information.

Additional information can be found at w3schools.com

Block and Inline Elements

HTML elements are either inline or block. Block elements form an "invisible box" and divide the documents into sections. Inline elements only occupy the space bounded by the tags of the inline element.

Lesson 2: Creating a Structured Document with HTML

Google Chrome Developer Tools

The Google Chrome browser contains built in development tools that can be accessed through a menu or by entering in Ctrl+Shift+i. The tools will help greatly with be able to understand how the browser is interpretting your code. Additional details cane be found at the Google Chrome DevTools Overview page.

HTML Tree Structure

The tree like structure comes from a tree / parent child relationship that creates a logical structure within documents. The Document Object Model (DOM) is an API for markup documents that helps create the tree structure.

Webpage Layout Structure

Webpages when built using HTML contain box structures in order to create a design layout. Many box structures can be used within one another in order to create as detailed of a layout as required. These boxes are created within the tree strcture detailed previously.

Sublime and Code Editors

Programmers typically use special editors when writing code. These editors assist by color coding different tags and auto completing a closed tag when an opening tag is generated. Sublime is the most popular code editor but many other options exists as well such as Notepad++. Progamming without an editor is possible but not recommended.

Leason 3: Adding CSS Style to HTML Structure

CSS

CSS stands for Cascading Style Sheets and it defines how HTML elements are to be displayed. A style sheet is added by inserting the line <link rel="stylesheet" type="text/css" href="nameofstylesheet.css"> at the top of your HTML document. Style can be inherited through the parent tree structure, so the page design needs to take that into account. Style should not be set in the HTML document but rather in the CSS file.

Avoiding Repetition

CSS greatly reduces the amount of repititive code in a document by allowing style elements to be grouped and reused throughout the document. With common pieces of code being used for style throughout a document, it allows a great amount of flexibility where the overall site design can be changed rapidly without having to modify individual elements in HTML.
